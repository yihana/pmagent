KBS 지능적 미디어 속성 추출용 데이터 관리시스템 개발 RFP
# **용역과제 제안요청서(RFP)**

# **과제명**

지능적 미디어 속성 추출용 데이터 관리시스템 개발

# **개발기간**

계약 후 4개월

(발주기관 사정에 따라 조정 가능)

# **주요개발내용**

- 지능적 미디어 속성 추출을 위한 데이터 관리 도구 개발
- 지능적 미디어 속성 백엔드 시스템 개발

※ 본 용역의 모든 결과물의 권리는 KBS에 귀속

# **제출서류**

**1) 입찰등록 서류**

- 입찰참가신청서
- 입찰보증금 지급각서
- 가격입찰서 + 산출근거
- 보안준수서약서
- 최근 5년 이내 관련 연구개발 실적
- 기타 요청 서류

**2) 제안서류**

- 개발제안서 제출 공문
- 개발제안서 13부(원본 3, 사본 10)

# **자격요건**

- 「국가계약법」 제76조 부정당업자 제외
- 최근 5년 이내 관련 연구개발 실적 보유
- 콘소시엄의 경우: 2기관 이내, 기관별 역할 및 가격 명기

# **II. 용역과제 세부사양**

※ <내용기반 검색 시범서비스 개발 개요> 도식 포함

# **1. 데이터 관리 도구 개발**

**기능 요구사항**

1. CRUD 기능(생성/읽기/갱신/삭제)
2. 대용량 콘텐츠/메타데이터 일괄 업로드·다운로드
3. AI 엔진 추출 결과 ↔ 서비스 메타데이터 매핑
4. AI 기반 태깅 도구

**개발 유의사항**

- 사용자 친화적 UI/UX
- 업로드 대상: KBS 오픈아카이브 & ETRI 시스템
- AI엔진 결과는 ETRI 사물 속성 모듈 활용
- 메타데이터 설계는 착수 후 6주 내 KBS·ETRI 협의

# **2. 백엔드 시스템 개발**

**기능 범위**

- 오픈아카이브 콘텐츠 저장
- 시범서비스용 메타데이터 저장
- AI 속성 추출·장면 판별 결과 저장
- 확장된 미디어 속성 DB
- AI엔진 연동 모듈 및 스케줄러

**개발 유의사항**

- KBS 아카이브 규격 준용
- AI엔진은 ETRI 모듈 사용
- 진행상황 따라 모듈이 미제공 시 오픈소스 등으로 구현 필요

# **III. 최종 결과물**

- 데이터 관리 도구(SW, 소스코드)
- 백엔드 시스템(SW, 소스코드)
- 시스템 설계서(관리DB 포함)
- 최종 보고서
- 사용자 매뉴얼
- (검수는 KBS 지정 HW에 설치해 수행)

# **IV. 수행 지침**

- 계약 후 7일 이내 수행계획서 제출
- 월 1회 추진협의체 회의
- 문제 발생 즉시 발주자 보고
- 인력 변경 시 승인 필요
- 결과물의 모든 권리는 KBS 귀속
- 보안 엄수: 자료 저장/반출 금지, 외부 유출 금지
- 납품은 검수 합격 후 설치
- 무상보증 1년

# **V. 선정 및 평가**

# **1. 입찰 자격**

- 최근 5년 이내 관련 실적 보유 업체
- 기술개발·인증·납품 실적 제출

# **2. 입찰 절차**

입찰공고 → 현장설명회 → 제출 → 기술평가+가격입찰 → 계약

**평가 기준**

- 기술평가 100점
    - RFP 부합성
    - 추진계획
    - 기대효과
    - 인력 전문성
    - 유사 실적 평가
- 

# **VI. 제안서 작성 요령**

- 한국어 작성
- 모든 약어는 풀어서 표기
- 모호한 표현 지양(“가능하다”, “고려한다” 등)
- 포함 내용: 개발내용·일정·전략·예산·인력 등
- 제안서 내용은 계약조건으로 간주
- 허위 시 계약 해지 가능
